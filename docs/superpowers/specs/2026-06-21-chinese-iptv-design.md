# IPTV 中文频道订阅设计

## 目标

从 `https://iptv-org.github.io/iptv/languages/zho.m3u` 生成可直接订阅的中文版本，并通过 GitHub Actions 每日自动同步。范围包括上游标记为中文的中国大陆、港澳台及海外频道。用户可见的频道名称、频道分组、清晰度和状态标签除 `CCTV` 品牌外不得残留英文；所有 CCTV 频道统一归入 `CCTV` 分组。技术标识、台标和直播地址保持上游原值。

## 方案

仓库保存一份确定性的频道 ID 到中文名称映射。更新脚本下载上游 M3U 和 iptv-org 频道数据库，优先使用人工审校映射，其次使用数据库中文别名；清晰度、状态标签与分组由固定规则翻译。若下载失败、格式异常、频道缺少中文名称或输出的可见字段仍含英文字母，脚本以非零状态退出，保留已发布文件。

`iptv/zho.m3u` 是发布产物。`.github/workflows/update-iptv.yml` 每天 UTC 22:20（北京时间次日 06:20）执行，也允许手动运行；只有内容变化时才提交并推送。

## 边界

- 不代理、缓存或转播视频。
- 不修改 `tvg-id`、`tvg-logo` 或直播 URL。
- 不声称产物是 iptv-org 官方中文订阅，文件头注明数据来源。
- 自动化不能猜测新频道译名；未知且无中文别名时安全失败，避免发布半中文列表。

## 验证

单元测试覆盖频道映射、分组翻译、清晰度和状态标签、未知频道失败、M3U 技术字段不变。集成验证使用实时上游生成产物，检查条目数量一致、所有流地址一致、可见字段无英文字母。
